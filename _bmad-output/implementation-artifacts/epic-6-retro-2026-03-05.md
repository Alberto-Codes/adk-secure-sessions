## What Didn't Go Well

### Stale claims propagate wider than expected
A single inaccurate phrase ("drop-in replacement") had copied itself into 6+ files across different categories (PyPI metadata, README, docs, docstrings, planning artifacts). "Independent of ADK" appeared in 4+ files. Copy-paste documentation is a virus — one wrong claim self-replicates.

Story 6.2 review found `.specify/memory/constitution.md` still had stale language — a governance file nobody thought to check. The team had a blind spot for non-obvious file locations.

### No automated markdown rendering validation
Docvet checks prose style, ruff checks Python, but nothing validates that markdown tables/lists render correctly after edits. Story 6.3's HIGH finding (blockquote markers breaking GFM table rendering) was caught only in adversarial review.

### Cross-cutting test selection is ad-hoc
Dev agents pick cross-cutting tests based on personal judgment during implementation. TEA agent (Murat) has risk-based testing expertise and should own the prioritization via a standing coverage gap document.

---

## Key Insights

1. **Docs-only stories still need adversarial review** — markdown rendering issues and stale-claim propagation are real defects, not cosmetic concerns
2. **Copy-paste documentation self-replicates** — grep verification after every documentation change is essential to prevent stale instances
3. **TEA should own cross-cutting test prioritization** — systematic risk-based selection over ad-hoc dev picks
4. **Story-to-story learning chains accelerate delivery** — explicit "Previous Story Intelligence" sections in each story document made patterns transferable
